1. What are the differencs betwwen Local Storage vs Session Storage?

Local Storage

a. The keys and the values stored with localStorage are always in the UTF-16 string format.

b. Local storage maintains data across all instances of a site, whether they're in different tabs or windows.

c. It is also permanent, so the data won't disappear when you close your browser.

Session Storage

a. Opening several tabs/windows with the same URL creates sessionStorage for each tab/window.

b. Duplicating a tab copies the tab's sessionStorage into the new tab.

c. Closing a tab/window ends the session and clears objects in sessionStorage.


2. What are the difference between Global Scope vs Block Scope?

Global Scope

a. A variable exists inside or outside a block is called a Global Scope variable.

b. Global Scope variable can be declared outside all functions or even curly braces ({}).

c. The global variables can be accessed by any line of code in the program, including inside blocks.

Block Scope

a. The block scope can be defined as the scope of the variables inside the curly brackets {}.

b. The curly brackets can be of loops, or conditional statements, or something else.

c. The valiable of a block scope can be accessable only within the curly brackets {}.


3. How JavaScript Event Loop works?

Event Loop

a. JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.

b. The event loop is a constantly running process that monitors both the callback queue and the call stack.

c. If the call stack is not empty, the event loop waits until it is empty and places the next function from the callback queue to the call stack.

d. If the callback queue is empty, nothing will happen.


4. What are ways of getting Undefined in JavaScript?

a. A variable that has not been assigned a value is of type undefined.

b. A function returns undefined if a value was not returned.

c. A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value.
